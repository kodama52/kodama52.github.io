Southern California Edison: Website Redesign

The Southern California Edison Website Redesign project is on-going and stems from an overwhelming amount of calls to their help center as well as a brand refresh. The goal is to reduce the help calls by 370,000/yearly, design uniform experiences, and update siteâ€™s overall image.

Project Overview and Role

I am working within a team of 9 with 5 other designers in order to redesign the Southern California Edison website for both residential and commercial use. My specific tasks are as follows:

  • Design a Registration flow for all SCE users: Map out user journeys. Create high-fidelity prototypes and collaborate with developers to implement the experience for both desktop and mobile environments.
  • Develop reusable UX components with user flows and wireframes: Collaborate with other UX Designers to ensure consistency of design and experiences.
  • Conduct UAT on all developed components: Conduct UAT on all developed components: Review live builds against wireframes and design specs to identify defects or usability concerns. Document findings and coordinate with dev team to resolve issues.
  • Lead UAT and PIV testing for Multi-Factor Authentication (MFA): Collaborate with QA and development teams to plan and execute User Acceptance Testing (UAT) and Product Improvement Validation (PIV) sessions. Ensure the MFA experience is free of defects.
  • Organize and categorize FAQs: Create a structured and searchable FAQ experience by topic, reduce all duplicated questions, and create a usable SVG file to pull from.

Technical Details & Challenges

Core Challenges

  • State Synchronization: Managing real-time updates across multiple components (product listings, cart icon, checkout form) efficiently without causing re-renders. Solved using `useReducer` and the Context API.
  • API Security: Implementing secure JWT (JSON Web Token) authentication for user sessions and protecting sensitive API routes like inventory management.
  • Responsive Design: Ensuring the complex product grid and checkout forms maintained perfect usability and aesthetic appeal across mobile, tablet, and desktop viewports using a mobile-first Tailwind CSS approach.

Technology Stack

  • Frontend: React, JavaScript (ES6+), Tailwind CSS.
  • Backend: Node.js with Express framework.
  • Database: MongoDB (NoSQL) for flexible data storage.
  • Deployment: Docker containerization (local), Netlify/Heroku (production deployment).

Outcome

The final product is a fully functional, high-speed mock e-commerce site capable of handling 1000+ mock products. It achieved a Lighthouse performance score of 95+ on desktop view, demonstrating strong technical execution.
