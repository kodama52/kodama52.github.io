CATA Bus Mobile App

Overview

My Role

UX Designer, UX Researcher

Timeline

8 Weeks

Centre Area Transportation Authority, CATA, has a mobile application that lets their users check the bus schedule from specific locations. In theory this allows users to plan their commute very easily.

Though, the original CATA Bus application had one page to view all information regarding bus schedules. This created difficulty for the user to navigate since the application displayed excessive data than the user needed at any point in time. Which caused confusion and frustration for their user base as well as frequent application shutdowns due to network overload.


Project Focus

On my team I helped design and create a new and improved format of the current CATA Bus application. We decided to format that application with three main pages to organize information for the user. These pages display:

  • Bus route information based on user location instead of overloading the screen with information by showing the full schedules
  • Frequently used routes
  • Settings to further customize the experience

Approach

I worked in a team that maximize workflow with the agile framework. We could always pivot to adjust our designs to the new feedback we gathered for each discovery stage.

Developement

I conducted secondary user feedback via live surveys to gain a better understanding of how the prototype should be laid out. From this feedback I constructed several low fidelity prototypes for my team.

The high-fidelity prototype was designed and animated with Adobe XD once we received our last round of user test feedback. I added in a short interactive guide that steps the user through the learning process of using the new CATA Bus application.
